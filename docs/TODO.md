## Глобальные улучшения агента

1. [ ] Сделать агента stateful по знаниям о БД
   - Кэшировать валидные объекты/поля/связи и переиспользовать между сессиями.
   - Эффект: меньше «угадывания» и recovery-циклов.

2. [ ] Разделить стратегию на `fast-path` и `safe-path`
   - `fast-path`: минимум шагов для типовых запросов.
   - `safe-path`: обязательные проверки схемы/полей/доступов для сложных или неясных задач.
   - Переключение автоматически по риску запроса.

3. [ ] Ввести бюджет выполнения на уровне оркестрации
   - Лимиты не только на токены, но и на число циклов `Plan -> Execute -> Recover`.
   - При превышении — ранний stop с техничным, полезным summary.

4. [ ] Формализовать DSL как `contract-first`
   - Единый валидатор + нормализатор до execute.
   - Строгие версии контракта (`dsl_version`) и совместимость между версиями.

5. [ ] Укрепить стратегию деградации (`graceful degradation`)
   - Если «идеальный» путь не проходит, агент должен отдавать частично полезный результат, а не застревать.
   - Пример: «вот данные по ближайшему валидному объекту + что нужно уточнить для точного ответа».

6. [ ] Сделать observability как у production-сервиса
   - Трассировка по `trace_id`, метрики `p50/p95 latency`, `token cost per intent`, `recovery success rate`.
   - Дашборд причин деградации качества (а не только `pass/fail`).

7. [ ] Улучшить качество промптов через policy-layer
   - Не раздувать один огромный промпт, а собирать его из модульных политик.
   - Политики: безопасность, формат ответа, SQL/1C-style ограничения, anti-loop.

8. [ ] Развивать автотесты как систему качества, а не только gate
   - Добавить regression-набор «из боевых логов».
   - Делать дифф-проверку не только по `pass/fail`, но и по эффективности (токены, шаги, retries).

9. [ ] Стабилизировать RAG-контур

   - Связи по объектам метаданных (ссылочные реквизиты, граф зависимостей).
   - Единый лимит токенов на RAG-блок в промпте (сейчас: Top-5 × 300 символов, без общего ceiling).
   - Опционально: семантический поиск (embeddings + Qdrant/MCP) вместо только TF-IDF.

12. [ ] Рассмотреть вариант 2: отдельный локальный сервис `rag-retriever`
