# Тесты для ИИ Агента

Этот пакет содержит UI-тесты Vanessa Automation для формы ИИ Агента после рефакторинга интерфейса.

## Файлы

- `TestAIAgent.feature` - файл с тестами в формате Gherkin для Vanessa Automation
- `AttachFiles.feature` - UI-тесты на прикрепление файлов в форме агента
- `testdata\notes.txt` - тестовый текстовый файл для сценариев Vanessa
- `testdata\sample.csv` - тестовый табличный файл для сценариев Vanessa

## Описание тестов

### 1. Smoke-проверка открытия формы
- Открывает форму ИИ Агента
- Проверяет, что новая структура формы загрузилась полностью
- Проверяет наличие ключевых элементов: поле ввода, кнопка отправки, вложения, план, измененные объекты и decision timeline

### 2. Новый диалог сбрасывает состояние формы
- Создаёт новый диалог
- Проверяет очистку поля ввода
- Проверяет, что панели плана, изменённых объектов и decision timeline пусты
- Проверяет, что статус соответствует начальному состоянию нового диалога

### 3. Прикрепление файлов
- `AttachFiles.feature` проверяет прикрепление `txt` и `csv`
- Проверяется статус `Вложение добавлено`
- Проверяется появление системного сообщения о вложении в истории чата

## Запуск тестов

### Через Python-скрипт

Используйте скрипт `update_and_run_vanessa.py`, указав файл теста:

```bash
python automation/vanessa/update_and_run_vanessa.py --feature-file automation/vanessa/TestAIAgent.feature
python automation/vanessa/update_and_run_vanessa.py --feature-file automation/vanessa/AttachFiles.feature
```

### Прямой запуск через Vanessa

1. Откройте 1С:Предприятие в режиме тестирования
2. Загрузите обработку `vanessa-automation-single.epf`
3. Укажите путь к файлу `TestAIAgent.feature`
4. Запустите выполнение сценариев

## Требования

- Настроенное подключение к GigaChat API (в регистре сведений `ИИА_НастройкиПользователей`)
- Для `AttachFiles.feature` должны существовать файлы `testdata\notes.txt` и `testdata\sample.csv`

## Примечания

- Основной `TestAIAgent.feature` теперь не зависит от реального ответа LLM и используется как быстрый regression/smoke контур после UI-рефакторинга
- Сценарии используют тестовые экспортные процедуры формы, чтобы проверять текущее состояние элементов без привязки к устаревшей таблице сообщений
